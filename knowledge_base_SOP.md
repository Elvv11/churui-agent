# knowledge_base.json 维护操作手册（SOP）

> 适用版本：Agent v3.1 及以上
> 最后更新：2026年7月17日
> 适用人员：运营/研发团队成员，无需编程基础

---

## 一、这是什么

`knowledge_base.json` 是 Agent 的知识库配置文件。Agent 回答问题时，会从这个文件里找依据。

原来知识库写在网页代码里，改一个字都要动代码、重新部署。现在独立出来，用记事本就能改。

## 二、准备工作

### 2.1 你需要什么

- 一台能上网的电脑
- 一个文本编辑器（Windows 自带记事本就行，推荐用 [VS Code](https://code.visualstudio.com/) 免费、有高亮、不容易出错）
- GitHub 仓库的访问权限（目前是 Elvv11/churui-agent）

### 2.2 打开文件

1. 打开 https://github.com/Elvv11/churui-agent
2. 找到 `knowledge_base.json`，点击文件名
3. 点击右上角的铅笔图标（Edit this file）
4. 直接在网页上编辑，或者复制全部内容到本地编辑器改完再贴回来

## 三、文件结构速览

整个文件分为十个区块：

| 区块 | 字段名 | 存什么 | 更新频率 |
|:---|:---|:---|:---|
| 基本信息 | `company` | 公司名称、成立年份、所在地、专利 | 几乎不变 |
| 核心产品 | `products` | 四款自研产品的技术参数 | 产品迭代时更新 |
| 上游零部件 | `upstreamComponents` | 六大类零部件的供应商、品牌、型号 | 供应链变化时更新 |
| 竞品（国内） | `competitors.domestic` | 路远/易通/木几的型号和客户 | **经常变** |
| 竞品（国际） | `competitors.international` | ASM/松下/富士/雅马哈的市占率 | 季度更新 |
| 下游应用 | `downstreamApplications` | 七个行业的需求和工艺要求 | 市场变化时更新 |
| 市场数据 | `marketData` | 市场规模、国产化率、趋势 | 年报发布后更新 |
| 政策 | `policies` | 补贴政策和产业计划 | **时效性强** |
| 目标客户 | `targetCustomers` | 江浙沪客户采购数据和替代机会 | **经常变** |
| 行业资源 | `industryResources` | 展会展、媒体、数据源、招标平台 | **时效性强** |

## 四、常见操作

### 4.1 改一个已有的数据

比如路远的自研率从 98% 变成了 99%。

找到这一段：
```json
"name": "路远智能",
"website": "faroad.cn",
"selfDevelopRate": ">98%",
```

把 `">98%"` 改成 `">99%"`，保存。

### 4.2 新增一个条目

比如政策板块，国务院出了新政策"SMT 行业进口替代专项基金 200 亿"。

找到 `"policies"` 段落，在最后一条后面加逗号，然后新增：

```json
"policies": [
  {
    "name": "工信部2026设备更新补贴",
    "detail": "补贴15%，门槛100万，SMT贴片机属适配设备"
  },
  {
    "name": "江苏省智改数转三年行动计划",
    "detail": "SMT产线升级补贴最高500万"
  },
  {
    "name": "国务院SMT进口替代专项基金",
    "detail": "规模200亿，重点支持贴片机核心零部件国产化"
  }
]
```

注意：最后一条后面不加逗号，前面的每条后面都要加逗号。

### 4.3 新增一个竞品型号

比如路远出了新机型 LY-CP960。

找到 `"flagshipModels"` 段落，在数组里加一条：

```json
"flagshipModels": [
  {"model": "LY-CP860", "type": "高速泛用", "speed": "0.06s/元件", "accuracy": "±0.03mm"},
  {"model": "LY-BS860", "type": "磁悬浮直线驱动", "acceleration": "2.5G"},
  {"model": "LY-CP960", "type": "超高速泛用", "speed": "0.04s/元件", "accuracy": "±0.02mm"}
]
```

### 4.4 新增一个目标客户

比如新发现了一家潜在客户"博众精工"。

找到 `"targetCustomers"` 段落，在最后加：

```json
{
  "name": "博众精工",
  "note": "3C自动化设备龙头",
  "annualPurchase": "年采导轨+伺服~3000万",
  "opportunity": "正在推进国产替代，楚瑞滚珠滑轨可送样测试"
}
```

### 4.5 更新展会日期

展会每年日期可能变化。

找到 `"industryResources"` → `"exhibitions"`，修改对应展会的 `"schedule"` 字段即可。

### 4.6 删除一条过时数据

直接删掉对应的整个 `{ ... }` 块。注意删完之后检查逗号——每两个条目之间必须有一个逗号，最后一条后面不能有逗号。

## 五、保存后怎么生效

### 方式一：直接在 GitHub 网页上改（推荐）

1. 改完后点页面底部的绿色按钮 "Commit changes"
2. 在弹出的框里简单写一句你改了什么（如"更新路远自研率至99%"）
3. 点 "Commit changes" 确认
4. GitHub Pages 自动部署，等 1-2 分钟
5. 刷新 Agent 页面 https://elvv11.github.io/churui-agent/
6. 问一个相关问题，验证新数据是否生效

### 方式二：本地改完上传

1. 把 `knowledge_base.json` 下载到电脑
2. 用编辑器修改并保存
3. 上传到 GitHub 仓库替换旧文件
4. 等 1-2 分钟，刷新 Agent 页面验证

## 六、常见错误及排查

### 6.1 JSON 格式错误——Agent 加载失败

**症状**：刷新 Agent 页面后提问，发现回答质量变差、没有最新数据。

**原因**：JSON 格式写错了。通常是**逗号问题**——该加逗号的地方没加，或者最后一条多加了逗号。

**排查**：
1. 打开 https://jsonlint.com/
2. 把 `knowledge_base.json` 的全部内容粘贴进去
3. 点 "Validate JSON"
4. 绿色 = 格式正确；红色 = 有错误，它会指出第几行有问题
5. 根据提示修改后重新上传

### 6.2 改了但没生效

**可能原因**：
- GitHub Pages 还没部署完（等 2 分钟再刷新）
- 浏览器缓存了旧文件（按 Ctrl+F5 强制刷新）
- JSON 格式错误导致加载失败，Agent 用了兜底提示词（按 F12 打开控制台，看有没有红色报错）

### 6.3 逗号规则速记

```
正确：
[
  {"a": 1},
  {"a": 2},
  {"a": 3}
]

错误（最后多了一个逗号）：
[
  {"a": 1},
  {"a": 2},
  {"a": 3},
]

错误（中间少了一个逗号）：
[
  {"a": 1}
  {"a": 2},
  {"a": 3}
]
```

## 七、建议的维护习惯

1. **每次修改前先看一眼文件顶部 `lastUpdated` 字段**，确认上次更新时间
2. **改完后把 `lastUpdated` 改成当天日期**，方便团队其他人知道知识库的新鲜度
3. **每次只改一个区块**，改完就上传，不要攒一堆一起改——分开改出错了容易定位
4. **不确定的数据不要往里写**，知识库追求准确而非数量，宁缺毋滥
5. **竞品和客户信息涉及报价和商业判断，建议修改前和领导确认**
